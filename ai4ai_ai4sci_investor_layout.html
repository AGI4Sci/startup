01 · Goal

目标与边界

回答三件事：最终要建设什么、第一年验证什么、哪些资产会形成长期复利。

Company

长期目标

建设下一代科学研究基础设施：连接科学知识、计算工具、专家判断和实验反馈，支持 AI 在真实科研任务中持续改进。

  • AI 自进化目标：支撑 AI agent 多尺度递归改进：模型能力、科学记忆、agent harness（执行/记录/回放/评测框架）和科研实验环境共同进化。
  • 科学影响力目标：让 AI 参与真实科学发现：从文献理解、假设生成、实验设计到结果复盘，形成可复现、可审查、可发表的科研过程。
  • 产业影响力目标：把科研线索转化为产业方能判断、能行动的项目资产：靶点判断、候选设计、实验验证、对外合作、临床前申报或产品化决策。
Year 1

第一年验证

用 1 个生命科学项目 + 1 个关键 ML 任务验证三件事：AI 自进化能力、科学质量、产业价值。

  • AI 自进化验证：选择 1 个关键 ML 场景(例如长序列建模、蛋白结构预测、长文本 RL 算法)，在公开 benchmark 或可复现实验上达到 / 超过当前 SOTA
  • 科学验证：围绕 1 个 target 完成 Research Dossier，达到可向 Nature Communications 及以上期刊投稿的质量标准
  • 产业验证：完成 1 个 Target-to-Binder Sprint，8-12 周内生成 50-300 条候选，Top 20-50 条进入表达、结合和成药性验证
Asset

复利资产

Memory 是 source of truth，另外三类资产分别负责工具调用、科学判断和训练自进化。

  • Scientific Memory Engine：文献、数据库、claim-evidence-provenance、项目状态
  • Scientific Tool Engine：深度学习模型、计算工具、数据库 API、MCP 服务、wet-lab 接口
  • Scientific Review & Reward Engine：expert taste、rubric、wet reward、human override
  • AI-Ready Training Data Engine：SFT、preference、agentic RL、agent self-evolve dataset

02 · Users

目标用户

第一批用户不是泛科研人群，而是有明确研发项目、真实实验反馈和付费动机的 discovery team。

ICP

核心客户

20-200 人规模的 biotech / discovery group；已有 target、disease hypothesis 或 biologics 项目；有 wet-lab 或稳定 CRO 合作。

  • 内部有 1-5 个 computational scientist
  • 没有完整 AI platform team
  • 需要更快形成候选和证据包
Buyer

买方和使用者

经济买方是 CEO、CSO、VP Discovery、BD Head、Program Lead；日常使用者是 protein engineer、computational biologist、assay lead、translational scientist。

  • 买方关心项目进度、证据质量和融资/BD 叙事
  • 使用者关心候选排序、实验计划和复盘效率
  • 专家 reviewer 贡献 taste 和纠偏信号
Not First

暂不优先

泛科研助手、高校 PI 通用写作工具、大药企总部级平台采购，都不适合作为最早期验证主线。

  • 泛科研需求分散，数据闭环弱
  • 高校场景容易退化成 RAG / 写作
  • 大药企适合作 lighthouse pilot

03 · Product Offer

产品形态

第一阶段按“服务辅助的软件产品”售卖：客户买项目结果，我们沉淀平台模块、数据结构和训练资产。

Target-to-Binder Sprint

  1. 8-12 周项目包。
  2. 交付 Target Card、candidate set、expert ranking、wet validation plan/result review。
  3. 适合早中期 biotech、BD 前项目、PI 孵化项目。

Confidential Project Room

  1. 客户和内部团队共同使用的项目空间。
  2. 承载 target、evidence、candidate、experiment、review、decision log。
  3. 客户数据 no-training-by-default，后续升级 dedicated tenant / VPC。

Evidence + Dataset Package

  1. 给客户：BD / IND / paper-ready research dossier。
  2. 给内部：SFT、preference、critique、failure recovery、RL transition。
  3. 同一项目输出人类可读和机器可读两类资产。

05 · AI4Sci Closed Loop

垂直闭环

Target-to-Binder 是第一条可快速交付的闭环；其它场景沿同一证据链扩展。

证据链

每一步都要产生客户交付物和可回流数据。

1Target / hypothesis：evidence、assay feasibility、kill criteria
2Candidate design：binder / antibody / nanobody generation and ranking
3Wet validation：expression、purification、binding、developability
4Phenotype / safety：organoid、organ-chip、HCI、toxicity、mechanism reward
5Dossier：continue / kill、BD、IND、publication-grade evidence package
6Training：review、failure、reward、trajectory 编译为训练数据

第一年垂直产品

不是只做一个场景，而是用第一个场景启动闭环。

1Target Intelligence & Hypothesis Review
2Target-to-Binder Candidate Decision Package
3Binder Optimization & Developability Loop
4Organ-chip / Organoid Phenotype Reward Package
5BD / IND / Publication-grade Research Dossier

06 · Roadmap

第一年验证路线

路线图按两个任务组织：一个生命科学闭环，一个关键 ML benchmark。

0-3 个月

选题和基线

选定 1 个公开或低敏 target，以及 1 个关键 ML 场景；明确科学投稿标准、产业交付标准和 SOTA baseline。

  • Research Dossier outline + expert rubric
  • ML benchmark、baseline、harness v0
3-6 个月

生命科学闭环

围绕 1 个 target 生成 50-300 条抗体 / 纳米抗体候选，Top 20-50 条进入表达、结合和成药性验证。

  • Research Dossier + 候选物决策包
  • raw data、failure、negative samples 入库
6-12 个月

ML SOTA 验证

聚焦一个关键 ML 场景：长序列建模、蛋白结构预测、生物基础模型或长文本 RL 算法。

  • 公开 benchmark 或可复现实验达到 / 超过 SOTA
  • 沉淀 harness、训练数据和评测流程
12-18 个月

复用判断

判断 Research Dossier、候选物决策包和 ML harness 是否可复用到下一个项目。

  • Project Room / private deployment 进入销售判断
  • AI-ready training dataset v0

07 · Metrics

验收指标

指标对应三类目标：科学上能投稿，产业上能决策，AI 上能超过强基线。

North Star

1 个生命科学项目和 1 个关键 ML 任务，能否同时形成科学、产业和 AI 自进化的硬证据

科学验证Research Dossier 达到 Nature Communications 及以上期刊投稿准备标准；3 位外部专家盲审 ≥4/5。
产业验证8-12 周完成 Target-to-Binder Sprint；交付候选物决策包，明确继续优化、停止投入或下一轮实验建议。
AI 自进化1 个关键 ML 场景达到 / 超过当前 SOTA；沉淀可复用 harness、训练数据和评测流程。
Wet reward 质量90%+ 样本具备 metadata；包含 raw curve / raw image、QC flags、negative data、SOP version。

08 · Resource Ask

所需资源支持

资源请求围绕最小闭环：团队、试点客户、wet-lab network、数据权利。

核心团队

  1. 8-10 人：Agent、Memory、Review/Eval、Trajectory、蛋白设计、wet-lab interface。
  2. 2-3 位科学顾问 / expert reviewer。
  3. 1 位科研 workflow PM。

Wet-lab Reward Network

  1. Pilot A：antibody / nanobody validation。
  2. Pilot B：organoid / organ-chip phenotype reward。
  3. 候选伙伴：百英、金斯瑞、义翘、百普赛斯、KACTUS、冠科、Xellar、大橡、丹望。

数据与合作边界

  1. 必须拿到 raw data、negative data、metadata、SOP version、QC flags。
  2. 客户数据 no-training-by-default。
  3. 跨客户训练只在 opt-in 和脱敏条件下进行。

09 · To Discuss

需要继续讨论的决策

这些会影响第一年销售、产品和资源配置，不适合在页面里假装已经完全确定。

第一批客户优先级

建议先找早中期 biotech / discovery group；大药企做 lighthouse pilot。需要你确认是否接受较窄 ICP。

第一单包装

建议卖 Target-to-Binder Sprint，而不是直接卖平台订阅。需要确认是否以 project-based revenue 作为第一年主口径。

Wet reward 范围

建议第一阶段覆盖 binding / expression / developability；phenotype reward 作为第二个 pilot 接入。

数据权利默认值

建议客户数据默认不训练；schema、rubric、workflow、匿名统计和授权数据用于平台改进。

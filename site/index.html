01 · Goal

目标与边界

回答四件事：最终要建设什么、第一年验证什么、哪些资产会形成长期复利，以及第一阶段不做什么。

Company

长期目标

建设下一代科学研究基础设施：连接科学知识、计算工具、专家判断和实验反馈，支持 AI 在真实科研任务中持续改进。

  • AI 自进化目标：支撑 AI agent 多尺度递归改进：模型能力、科学记忆、agent harness（执行/记录/回放/评测框架）和科研实验环境共同进化。
  • 科学影响力目标：让 AI 参与真实科学发现：从文献理解、假设生成、实验设计到结果复盘，形成可复现、可审查、可发表的科研过程。
  • 产业影响力目标：把科研线索转化为产业方能判断、能行动的项目资产：靶点判断、候选设计、实验验证、对外合作、临床前申报或产品化决策。
Year 1

第一年验证

用 1 个生命科学项目 + 1 个关键 ML 任务验证三件事：AI 自进化能力、科学质量、产业价值。

  • AI 自进化验证：选择 1 个关键 ML 场景(例如长序列建模、蛋白结构预测、长文本 RL 算法)，在公开 benchmark 或可复现实验上达到 / 超过当前 SOTA
  • 科学验证：围绕 1 个 target 完成 Research Dossier，达到可向 Nature Communications 及以上期刊投稿的质量标准
  • 产业验证：完成 1 个 Target-to-Binder Sprint，8-12 周内生成 50-300 条候选，Top 20-50 条进入表达、结合和成药性验证
Asset

复利资产

Memory 是 source of truth，另外三类资产分别负责工具调用、科学判断和训练自进化。

  • Scientific Memory Engine：文献、数据库、claim-evidence-provenance、项目状态
  • Scientific Tool Engine：深度学习模型、计算工具、数据库 API、MCP 服务、wet-lab 接口
  • Scientific Review & Reward Engine：expert taste、rubric、wet reward、human override
  • AI-Ready Training Data Engine：SFT、preference、agentic RL、agent self-evolve dataset
Boundary

第一阶段边界

第一阶段不是卖通用科研助手或总部级平台订阅，而是用项目制 sprint 服务有明确决策窗口的 discovery team。

  • 客户与伙伴边界：直接付费客户和标杆试点客户属于产业界客户，目标是收入和产业影响力；科研机构主要服务学术影响力和高质量项目源；实验验证伙伴多是我们付费采购或重点共建的对象，目标是形成验证壁垒。
  • 产品边界：Target-to-Binder Sprint 是第一单；Project Room 是交付载体，也是 Science Agent OS 的产品雏形。
  • 市场边界：第一阶段只服务有明确靶点或候选分子的生物药早研项目，帮助客户在 8-12 周内决定继续投入、停止、对外 BD，或进入下一轮实验；不覆盖小分子全流程、临床开发、常规 CRO 外包和通用科研工具。

04 · Delivery System

项目交付系统

这一层回答客户项目如何被生产出来：从项目输入，到工作区推进，再到交付档案和实验数据包，最后沉淀成下一次可复用的流程。

项目输入

客户给出 target、疾病假设、候选分子、项目约束、成功标准和停止标准。第一步不是跑模型，而是把项目边界、数据权限和判断口径说清楚。

项目工作区

所有证据、候选、实验、专家意见和决策记录进入同一个 Web 工作区。客户看到的是项目进展和判断依据，不需要理解底层技术模块。

交付生成

按用户场景导出候选物决策档案、论文材料、实验与数据包、管理层摘要或内部试点评估材料，让项目结果能被继续使用。

项目沉淀

每个项目结束后留下模板、数据标准、失败案例、评审规则、实验反馈和复盘记录，成为下一次项目更快、更准的基础。

05 · Technical Architecture

平台技术架构

这一层回答技术壁垒在哪里：不是把模型接到聊天框，而是把科研记忆、工具编排、专家判断、实验反馈和可回放轨迹组织成持续改进的系统。

1
Scientific Memory Graph把文献、数据库、内部材料和项目状态整理成 claim-evidence-provenance 图谱，保留来源、版本和冲突证据
2
Scientific Tool Orchestration统一调度结构预测、序列设计、数据库 API、文献检索、内部模型和 wet-lab partner 接口，并约束输入输出格式
3
Review & Reward Ledger记录专家 rubric、候选排序、人工 override、QC flags、wet assay 结果和分歧意见，让“好结果”可追踪、可比较
4
Execution Trace & Replay每次工具调用、候选变更、证据引用、审批 gate 和数据快照都留下轨迹，支持审计、复盘和复现实验路径
5
Training Data Compiler把项目轨迹编译为 SFT、preference、critique、failure recovery 和 RL-ready transition，用于下一代科研 agent 训练
6
Secure Project Runtime提供项目隔离、权限审计、no-training-by-default、预算控制、安全边界和工具调用策略

技术特色

普通 RAG 只能回答问题，普通 CRO 只能交付实验，普通报告工具只能整理材料。我们的核心是把证据、工具、评审、实验反馈和执行轨迹放在同一套可训练、可复盘的科研运行系统里。

Memory证据图谱、项目状态和版本化科研记忆
Tool可组合、可约束、可复现的科学工具调用
Reward专家判断、质控标准和 wet reward 记录
Trace可回放轨迹和可训练的项目数据资产

06 · Vertical Loop Example

Target-to-Binder 闭环示例

Target-to-Binder 不是全部业务边界，而是第一条最适合跑通的闭环：周期短、交付清楚、wet reward 明确，能同时验证客户价值和平台复利。

一次项目怎么跑

从客户输入到决策交付，每一步都留下证据、版本和复盘材料。

1客户输入 target、疾病假设、候选物或项目约束，明确成功标准和停止标准
2系统整理文献、数据库、竞品、结构信息和内部材料，形成靶点证据图
3生成或筛选 binder、antibody、nanobody 或 format options，并给出风险标签
4专家评审候选排序、机制假设、成药性风险和实验可行性
5Top candidates 进入 expression、binding、developability 等小规模 wet validation
6输出继续、停止、BD、补实验或下一轮优化建议
7把失败案例、实验反馈、专家判断和工具轨迹回流到平台

为什么先从这里开始

它同时具备商业清晰度、实验反馈和技术复用价值。

1输入清楚：客户通常已有 target、疾病场景或候选分子
2周期可控：8-12 周能形成候选排序和小规模验证结果
3交付明确：客户能用结果做 continue、kill、BD 或下一轮实验决策
4反馈真实：expression、binding、developability 给出可训练的 wet reward
5可扩展：同一证据链可延展到 binder optimization、phenotype reward、BD / IND / publication dossier

07 · Validation Plan

第一年验证计划

每个阶段只保留要完成的事和通过标准，用来判断科学、产业和技术复利是否同时成立。

0-3 个月

选题和基线

选定 1 个公开或低敏 target，以及 1 个关键 ML 场景；明确专家、候选实验伙伴、数据边界和 baseline。

  • 通过标准：项目边界、专家 rubric、实验伙伴 shortlist 和数据字段确认
  • 通过标准：ML benchmark、baseline、harness v0 可复现
3-6 个月

技术基线与实验准备

围绕 1 个 target 先完成 dry-run：证据整理、候选生成、专家评审和候选排序；同步完成实验伙伴筛选、SOP、样本表和数据回传模板。

  • 通过标准：生成 50-300 条候选，Top 20-50 条完成专家评审和实验优先级排序
  • 通过标准：wet-lab partner、SOP、QC 标准、metadata schema 和数据回传流程确认
6-12 个月

跑通湿实验闭环

Top candidates 进入表达、结合和成药性验证；同时推进关键 ML 场景，检查项目轨迹是否能变成可训练、可评测、可复用的数据资产。

  • 通过标准：交付候选物决策档案，给出继续、停止、BD 或补实验建议
  • 通过标准：raw data、negative samples、QC flags、失败原因和 wet reward 入库
  • 通过标准：关键 ML benchmark 达到强基线 / SOTA，并沉淀 harness 和评测流程
12-18 个月

判断可复制性

判断项目工作区、交付档案、实验数据包和底层 harness 是否能复用到第二个客户或第二个项目。

  • 通过标准：第二个项目启动成本显著下降，模板和数据标准可复用
  • 通过标准：形成可销售的项目工作区 / 私有部署判断

08 · Resource Ask

所需资源支持

按衔远当前阶段，建议采用 5200 万元 / 年验证版：用小而完整的核心团队，配常态 64 张 H200，把第一年重点压到 1 条可收费项目闭环、1 个关键 ML benchmark 和必要的外部实验验证上。

核心团队

  1. 全职员工：第一年 6 HC，覆盖 AI4Sci 负责人、ML research、训练评测、平台工程、数据工程、蛋白 / 抗体科学或产品交付。
  2. 实习生 / 访问学生：10-14 HC，主要支持 benchmark、数据清洗、文献证据整理、评测、实验数据标注和项目材料整理。
  3. 组织方式：顾问、IP 和合作成本不单列预算，必要工作并入全职团队职责、实验项目合同或数据库 / 软件采购。

算力 / 存储 / API 预算

  1. GPU：常态按 64 张 H200 等效卡 / 月配置；按 8 卡机器 8.8 万 / 月计算，基础年成本约 845 万人民币。
  2. 峰值额度：关键 benchmark 或候选生成窗口通过短租、排队和合作方资源补到 128-160 张 H200 等效卡 / 月。
  3. CPU / 内存：800-2,000 vCPU 等效资源池，3-8TB RAM；峰值阶段预留 8,000-20,000 vCPU、24-64TB RAM。
  4. 存储：80-150TB 热对象存储，50-100TB 高性能 scratch / feature store，0.5-1.5PB 对象 / 冷归档。
  5. 模型 API：通用 LLM、embedding、rerank、多模态和代码辅助，按全年 800 万人民币预算池管理。

科学软件与数据库

  1. 文献 / 专利：1-2 个核心数据库或 API，用于 prior art、竞争格局和 claim 边界确认。
  2. 蛋白 / 结构：UniProt、PDB、AlphaFold DB、抗体结构和 developability 相关工具链。
  3. 药物 / 靶点：药物、靶点、适应症、临床和交易数据库，支持 BD 与项目评估。
  4. 商业 license：按 2-4 个核心 seat 预留，优先采用项目制采购，避免中途因权限或额度卡住。

实验验证网络

  1. Pilot A：2 家 antibody / nanobody expression、binding、developability 验证伙伴。
  2. 第一年按 Top 20-30 条候选进入小规模验证预留实验额度，实验预算约 1000 万人民币。
  3. Pilot B：1 家 organoid / organ-chip phenotype reward 伙伴，只做小规模方法验证，不作为第一年刚性产能。
  4. 候选伙伴：百英、金斯瑞、义翘、百普赛斯、KACTUS、冠科、Xellar、大橡、丹望。

09 · Decision Defaults

建议直接拍板的执行方案

第一阶段不把问题继续摊开，而是先设定一组可执行默认值：用项目收入买时间，用 wet reward 买真实反馈，用 Project Room 沉淀平台雏形。

第一批客户优先级

先打 ADC / 双抗 / TCE / biologics 项目型 biotech，典型 20-200 人、有明确 target 或 candidate、有 8-12 周 continue / kill / BD 压力。大药企做单项目标杆试点，不作为第一收入主线。

第一单包装

首单卖 Target-to-Binder Sprint：候选生成、证据整理、专家评审、小规模 wet validation 和最终决策档案。Science Agent OS 只是交付底座，等 2-3 个项目后再转平台订阅、私有部署或企业 workflow。

Wet reward 范围

第一阶段只覆盖 expression / binding / developability，保证周期短、口径清楚、能形成候选排序。Phenotype reward 单独作为第二个 pilot，不塞进第一条闭环。

数据权利默认值

客户原始数据、项目材料、序列和实验结果默认不用于训练。schema、rubric、workflow、匿名统计、授权数据和内部生成轨迹可用于平台改进，销售和法务一开始就按这个边界沟通。

资源执行版本

第一年按 5200 万元验证版推进，而不是按重资产部门配置启动。先用 6 个核心全职、10-14 名实习 / 访问学生、常态 64 张 H200 等效卡和 2 家 wet-lab partner 跑通闭环；付费项目、实验回流和复用成本成立后再升级。

第一个 target 选择

选择公开证据较多、结构或抗体设计空间明确、IP 不过度敏感、8-12 周能看到 expression / binding 信号的 target。暂时避开临床路径复杂、依赖高敏内部数据或验证结果难讲清的项目。

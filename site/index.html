01 · Goal

目标

长期目标是建设下一代科学研究基础设施，支持 AI 持续递归进化，从而加速癌症等疾病的解决。

AI4AI

AI4AI 目标

把 AI4AI 底座做成可运行、可复盘、可迭代的系统。它不是单点工具，而是一套能组织复杂专业任务的基础设施。

  • 连接对象：科学知识、计算工具、专家判断、实验反馈和项目决策。
  • 核心能力：Agent Harness & Evolution、Scientific Memory Infrastructure、Scientific Environment Interface、Hierarchical Reward & Benchmark Interface、Training Data Compiler 和 Model Evaluation & Improvement。
  • 验收方式：能把任务计划、科学记忆、环境接口、reward / benchmark、实验结果和训练样本整理成一条可回放链路。
AGI4Sci

AGI4Sci 目标

把蛋白质设计和肿瘤治疗作为第一批真实科研任务，用外部合作项目检验 AI4AI 底座，并把反馈带回平台迭代。

  • 合作方向：优先联系蛋白质设计、肿瘤 biologics 和外部实验验证伙伴，形成候选项目池。
  • 牵引方式：用真实 target / candidate、专家判断、实验结果和失败样本暴露底座短板。
  • 项目验收：选择 1 个真实共创项目，完成候选生成、排序、专家评审和小规模实验反馈。

02 · Sci Partners

Sci 合作对象

这些伙伴的作用，是提供真实问题、专家判断和实验反馈。第一年先建立合作池，不把所有对象都写成收入主线。

项目型 biotech：优先联系

优先找肿瘤 biologics / protein therapeutics 早研团队。它们通常有明确 target、疾病假设或候选分子，也有 8-12 周继续 / 停止 / 合作决策压力，适合作为真实牵引。

03 · Progression

AI4AI 与 AGI4Sci 的递进关系

主线不复杂：AI4AI 先跑起来，Sci 项目再进来压测，项目结果最后回到 AI4AI。具体单子后续和生物公司谈。

Step 1

先把 AI4AI 跑通

先把任务计划、科学记忆、环境接口、reward / benchmark、数据整理和回放评测做成可运行系统。

  • 解决的问题：没有可复用底座，Sci 项目容易变成一次性咨询或外包交付。
  • 第一阶段产物：项目空间、工具注册、审批节点、执行回放、评测样本和训练数据格式。
Step 2

并行找 Sci 伙伴

平台建设期，同步联系蛋白质设计、肿瘤 biologics、类器官 / 器官芯片和实验验证伙伴，形成真实项目池。

  • 解决的问题：内部 benchmark 不够，需要真实专家、真实实验约束和真实失败样本。
  • 第一阶段产物：候选合作方、项目选择标准、数据授权边界、实验接口和专家评审规则。
Step 3

用一个项目做压力测试

底座可运行后，选择 1 个外部伙伴共创项目。优先方向是蛋白质设计 + 肿瘤治疗；类器官或器官芯片作为后续更硬的反馈接口。

  • 项目标准：有明确 target 或 candidate，实验周期可控，IP 不过度敏感，能在 8-12 周看到表达 / 结合 / 成药性信号。
  • 交付形态：付费 Sprint、共研试点或标杆项目都可以，但目标是验证底座，不是先把商业模式写死。
Step 4

把反馈带回底座

项目结束后，把证据链、工具轨迹、专家分歧、实验结果、失败样本整理成下一轮训练和评测材料。

  • 解决的问题：底座的成长需要真实反馈，而不只是离线报告或人工反复改任务说明。
  • 经营意义：如果 2-3 个项目都能复用同一套流程，再讨论项目室、私有部署、共研权益或分成。

05 · Sci Validation

Sci 验证体系

Sci 项目不是单次外包实验，而是分层 reward 环境。先用计算和高通量实验降低候选数量，再把少数候选送入类器官、肿瘤切片和器官芯片，形成可回流的训练与评测数据。

Principle

低成本层负责探索

计算数据、构型预筛和 2D 功能实验负责快速发现明显不合格的候选，让昂贵实验只服务少数高价值问题。

  • 目标：把几百到几千个候选压缩到几十个。
  • 数据：靶点表达、候选构型、基础杀伤、细胞因子和正常细胞安全信号。
  • 回流：形成早期 reward 模型和下一轮实验优先级。
Translational

中间层负责临床相关性

多 donor、肿瘤微环境压力、患者来源类器官和肿瘤切片，用来判断候选是否只在简单培养皿里好看。

  • 目标：识别 donor 变异、微环境失效、空间浸润不足和患者样本差异。
  • 数据：扩增、耗竭、低氧/TGF-β/CAF/TAM 压力、3D 浸润和残余克隆。
  • 回流：校准早期 killing、cytokine 和 TME reward 权重。
Risk

高保真层负责去风险

器官芯片不是主筛选平台，而是少数候选的 IND 前风险测试，重点看器官环境、动态暴露和安全窗。

  • 目标：回答是否存在肝毒性、内皮激活、巨噬细胞炎症放大和剂量窗过窄。
  • 数据：肝细胞、胆管、肝内皮、Kupffer/巨噬细胞、灌流芯片和剂量暴露读数。
  • 回流：形成 go/no-go 依据和高保真安全 reward。

层次化验证漏斗

这张表用于定义每层在 Sci 项目里的作用、通量、时间成本和计算替代空间。原则是：能计算预筛的先计算，必须湿实验确认的再实验，越往后候选越少、保真度越高。

层级 作用 典型通量 / 时间 湿实验属性 回流到 AI4AI 的资产
L0 计算与患者数据 判断靶点、疾病人群和正常组织风险是否值得投入。 10³-10⁵ 个假设；数天到 2 周。 主要可计算，少量病理或表达验证。 靶点风险图谱、患者分层、正常组织风险清单。
L1 候选结构设计 从大量分子或细胞治疗构型中排除明显不合格设计。 10²-10⁴ 个构型 / 轮；1-3 周。 可大量计算预筛，但表达和自发激活必须实验确认。 构型特征、预筛排序、失败构型样本。
L2 2D 功能筛选 测基础杀伤、结合、激活、细胞因子和正常细胞安全窗口。 30-300 个候选 / 轮；1-2 周。 必须湿实验，计算用于选点和预测曲线。 早期 reward、剂量/时间曲线、cytokine 风险标签。
L3 多 donor / 制造鲁棒性 判断候选是否跨 donor 稳定、可制造、不过早耗竭。 5-30 个候选 × 6-20 donor；3-6 周。 必须湿实验，计算用于批次异常预警和 donor 风险预测。 制造成功率、donor 变异、产品质量标签。
L4 肿瘤微环境压力测试 测试低氧、TGF-β、CAF/TAM、基质屏障等压力下是否仍有效。 5-20 个候选 × 4-10 条件；2-4 周。 必须湿实验，计算用于挑选最有信息量的压力条件。 失效机制、增强模块需求、TME reward。
L5 类器官 / 肿瘤切片 验证患者来源 3D 肿瘤里的空间浸润、杀伤和残余克隆。 3-10 个候选 × 5-30 样本；2-8 周。 必须湿实验，计算主要用于图像分析和患者响应分群。 空间杀伤图、患者异质性、残余克隆风险。
L6 肝安全 / 器官芯片 对 1-5 个候选做器官级安全去风险和剂量窗判断。 1-5 个候选 × 10-100 条件；4-12 周。 必须湿实验，计算用于剂量窗建模和风险排序。 IND 前 go/no-go、肝毒性/内皮/巨噬细胞安全 reward。
Organoid

类器官负责“像患者肿瘤”

类器官和肿瘤切片更适合看候选能不能进入真实 3D 肿瘤结构、是否覆盖患者异质性、治疗后是否留下低表达残余克隆。

Chip

器官芯片负责“像器官环境”

器官芯片更适合看血流/灌流、内皮屏障、肝脏免疫细胞、动态暴露和安全剂量窗。它不替代类器官，而是在少数候选上做高保真去风险。

06 · Validation Plan

18 个月验证计划

18 个月按能力成熟度推进，不按概念层层加码。每阶段都看三件事：平台能不能跑，Sci 合作有没有真实牵引，反馈能不能回到下一轮迭代。

0-3 个月

底座先跑起来

完成最小平台骨架，同时把合作方名单和项目选择标准建立起来。

  • 平台验收：项目空间、任务计划、权限、运行日志、回放评测上线。
  • 知识与工具：文献 / 专利 / 数据库接入；target / candidate / assay schema 定义；结构 / 序列工具进入 registry。
  • 反馈与数据：专家评审表、实验数据字段、授权边界、训练 / 评测样本格式确定。
  • Sci 进展：形成候选合作方 shortlist 和第一个项目筛选标准。
3-6 个月

完成内部 dry-run，选出真实项目

先用公开数据和合作方安全任务跑通闭环，再从项目池里选出第一个真实牵引项目。

  • 平台验收：从问题定义到候选生成、证据整理、专家评审、样本导出，全流程可回放。
  • 知识与工具：关键结论可追溯；候选生成、筛选、去重和风险标注可复现。
  • Sci 进展：和合作方完成 problem framing，明确 target / candidate、实验边界、数据授权和初步预算。
  • 数据回流：dry-run 产生第一批训练 / 评测样本，包含失败记录和人工修正。
6-12 个月

真实项目进入执行

第一个真实 Sci 项目进入候选生成、专家评审和小规模实验验证，重点看反馈能不能回到底座。

  • 平台验收：项目中的原始记录、失败样本、专家分歧和工具轨迹能进入数据整理管线。
  • Sci 进展：Top candidates 完成 expression、binding、developability；类器官 / 器官芯片视项目成熟度小范围接入。
  • 项目空间：实验 SOP、样本表、质控标记、实验状态回写到项目空间和 Scientific Memory Infrastructure。
  • 数据回流：形成首批授权范围内的训练样本、偏好样本、纠错样本和评测集。
12-18 个月

复用到第二个项目

接入第二个 Sci 项目，判断同一套底座是否能复用，并让启动成本、交付周期和人工组织成本明显下降。

  • 平台验收：第二个项目复用 runtime、memory、tool、ledger、compiler 和评测集，关键步骤不重写。
  • Sci 进展：复用 target intake、protein design workflow、专家评审规则、实验模板和决策档案结构。
  • 效率验收：新增项目启动时间、人工组织成本、交付周期明显下降。
  • 经营判断：再决定是否继续做项目室、私有部署、共研权益或资产分成。

08 · Company Value

一年成功后的公司价值

5200 万元验证版的资本意义，是把 AI4AI / AGI4Sci 从内部提案升级成可融资、可复用、可外部背书的公司级增长资产；如果证明 AI4AI 底座能被 AGI4Sci 的分子设计和器官芯片真实反馈持续训练，估值逻辑会高于普通项目交付、Copilot 工具或单一 AI 制药模型。

第一年投入 5200 万元

小而完整的核心团队、算力、模型调用、数据工程和一个真实 Sci 牵引项目。

基准融资能力 2-4 亿元

从概念验证进入“AI4AI 自我进化 + 科学发现加速”的 seed / pre-A 融资叙事。

基准估值空间 12-25 亿元

对应约 10-20 亿元可归因估值提升，前提是平台能被第二个项目复用。

一年成功后的重估结构

重估的理由很直接：如果第一年只完成一次科研交付，它还是项目服务；如果同一套底座能在分子设计、肿瘤 biologics 和器官芯片反馈中反复复用，并把每次结果沉淀成评测和训练资产，它就会被看作 AI4AI / AGI4Sci 平台。海外十亿美元级种子轮有明显的创始人溢价，不能照搬；更可比的是 AfterQuery、Deeptune、Mirendil 这类围绕环境、数据、评测和研发自动化形成估值的公司。

情景 一年执行结果 潜在新增投资 潜在估值 / 估值提升
保守成功 平台跑通，1 个真实项目完成专家评审和初步实验反馈，但第二项目复用和付费线索仍弱。 1-2 亿元 8-15 亿元；从内部项目升级为可融资平台雏形
基准成功 1 个真实 Sci 项目闭环，形成授权 trajectory / eval / reward 数据；第二个项目能复用同一套 workflow。 2-4 亿元 12-25 亿元；约 10-20 亿元可归因提升
强成功 Top candidates 出现实验信号，2-3 个合作方进入项目池，并沉淀可对外展示的 AI4AI × AGI4Sci Scientific Reward Environment。 4-8 亿元 25-50 亿元；进入 AI4AI / AGI4Sci 平台公司估值区间
极强成功 获得头部药企 / biotech 或 AI 基础设施资本背书，形成多项目复用、付费项目、权益分成和训练数据飞轮。 8-15 亿元+ 50-100 亿元+；需要外部强背书和多个真实 reward 闭环支撑

09 · Benchmarks

相关对标创业公司

对标不只看 AI4AI，也要看 AGI4Sci 生物口。本项目的叙事同时落在两边：一边是自我进化的 AI 底座，另一边是分子设计、疾病模型和器官芯片带来的真实科学反馈。
